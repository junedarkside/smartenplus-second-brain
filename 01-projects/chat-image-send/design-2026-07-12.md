# Chat Image-Send — Design Report (2026-07-12)

**Status:** DESIGNED — ready to implement. Feasibility ✅.
**Scope:** FE + AD + BE + Supabase. Image send for guest / OTA / logged-in users; admin sees + sends.
**Constraints honored:** no tech debt, no over-engineering, reusable existing utilities, zero breaking changes.
**Method:** 3 Explore agents (FE chat, AD chat, BE cs+Supabase) + 1 Plan agent design review.

---

## Architecture (verified 2026-07-12)

- **Message = text-only everywhere.** Django `cs.Message.body` (TextField); Supabase `cs_messages`: id, conversation_id, body, sender, created_at. No attachment field anywhere.
- **Two transports:** realtime (`NEXT_PUBLIC_CHAT_REALTIME=true` — client inserts direct to Supabase w/ Django-minted JWT, RLS-scoped) and polling (Django `POST /api/cs/messages/send/`). Sync task `cs/tasks.py:190-240` pulls Supabase rows back to Django, dedup by unique `Message.supabase_id`.
- **Auth tiers (all reuse existing logic):** staff session · logged-in Bearer · guest `X-CS-Guest-Token` · OTA = guest token minted from `ota_token` (identical downstream path).
- **Reusable pieces:**
  - BE: `dialogue/utils.py process_review_image()` — ext whitelist (jpg/png/webp/heic/heif), 20MB max, WebP ≤120KB, uuid name. `pillow_heif` registered globally (`dialogue/apps.py:16`) → server decodes HEIC. S3 via django-storages (`DEFAULT_FILE_STORAGE`, public-read).
  - AD: `components/utils/imageHelpers.js` (isHeic, convertHeicToJpeg), FormData RTK mutation pattern (`operatorsApi.uploadOperatorImages`).
  - FE: heic2any pattern exists (RateAndReviewForm) but **NOT needed** — server converts.

## Design — single nullable column, one endpoint, server-side upload

Add `image_url` (nullable) to Django `Message` + Supabase `cs_messages`. One new endpoint `POST /api/cs/messages/send-image/` (multipart: conversation, image, optional body):

auth → `process_review_image()` → S3 `cs_chat/` → reopen-check + Django Message.create (atomic) → Supabase mirror insert (service role) → backfill `supabase_id` → 201

**Both transports converge server-side for images.** Django's Supabase insert fires postgres_changes → realtime clients receive instantly, free. Polling clients get it via MessageSerializer. Text path byte-identical, untouched.

**Decisions:** 1 image/message · optional caption via body · image-only (empty body) allowed · 20MB input max · WebP ≤120KB output · throttle `cs_image: 30/hour`.

## 5 Critical corrections (from Plan-agent review — must implement)

1. **Duplicate-sync bug:** endpoint must capture Supabase row id (`Prefer: return=representation`) → backfill `Message.supabase_id` immediately, else `sync_chat_messages` re-imports the row as a duplicate bubble. Millisecond race window remains (accepted — worst case one dup).
2. **No client-side HEIC conversion:** server decodes HEIC via pillow-heif. FE preview = filename chip fallback; AD may reuse `convertHeicToJpeg` for preview only.
3. **Server-side preview fallback:** `get_last_message_preview` (`cs/serializers.py:44`) must return '📷 Photo' for image-only messages (polling inbox path) + client-side patch in AD `SideList.js:132` (realtime path).
4. **Reopen-check BEFORE Supabase insert** — else realtime clients see broadcast for a message that then 429s.
5. **Spoofed image_url via RLS:** customer-scoped realtime client could insert arbitrary `image_url` string. Mitigation: render-time S3-domain allowlist (`isAllowedChatImageUrl`) in FE + AD. Optional SQL CHECK constraint.

## Edge cases (all handled)

| Case | Handling |
|------|----------|
| Image-only, empty body | Allowed; body=''; preview '📷 Photo' server+client |
| Caption >4000 | 400 BODY_TOO_LONG (same as text) |
| Auto-reopen on image | Shared atomic helper; 429 decided before broadcast; orphan S3 file on 429 accepted |
| Sender attribution | Server-derived via shared helper — spoof-proof, identical to text |
| Sync back-fill | supabase_id backfilled → ignore_conflicts dedup; sync maps image_url for race window |
| Mark-read / unread | Untouched — ordinary rows |
| Realtime payload | Column auto-included; both realtime hooks' explicit field maps updated (only silent-drop point) |
| Supabase insert fails | 201 + warn; realtime viewers see on next history load |
| HEIC iPhone | Server decodes; FE preview = filename chip |

## Rollback

Every step independently deployable + backward-compatible. Rendering presence-guarded → revert order-free. Columns nullable additive — can stay forever. No data migration to unwind.
