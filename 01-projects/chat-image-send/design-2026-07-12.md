# Chat Image-Send — Design Report v2.1 (2026-07-12, 3-agent audit applied + guest gate)

**Status:** DESIGNED + AUDITED — ready to implement.
**Scope:** FE + AD + BE + Supabase. Image send for **OTA + logged-in users only**; plain guests text-only; admin sees + sends.
**Constraints honored:** no tech debt, no over-engineering, reusable existing utilities, zero breaking changes, image ≤80KB.
**Method:** 3 Explore + 1 Plan agent (v1) → 3 expert audit agents (code-reviewer BE, nextjs-architect FE/AD, backend-architect storage) verified every claim against live code (v2).

---

## Architecture (verified 2026-07-12)

- **Message = text-only everywhere.** Django `cs.Message.body` (TextField, non-nullable — empty string OK); Supabase `cs_messages`: id, conversation_id, body, sender, created_at. No attachment field anywhere.
- **Two transports:** realtime (`NEXT_PUBLIC_CHAT_REALTIME=true` — client inserts direct to Supabase w/ Django-minted JWT, RLS-scoped) and polling (Django `POST /api/cs/messages/send/`). Sync task `cs/tasks.py:190-240` pulls Supabase rows back to Django, dedup by unique `Message.supabase_id`.
- **Auth tiers (all reuse existing logic):** staff session · logged-in Bearer · guest `X-CS-Guest-Token` · OTA = guest token minted from `ota_token` (identical downstream path).
- **Reusable pieces:**
  - BE: `operators/utils.py process_operator_image(max_output_size, max_dimensions)` — parameterized (verified, no circular import cs←operators). Accepts jpg/jpeg/png/webp/gif/heic/heif/avif, 20MB max, WebP output. `pillow_heif` registered in `dialogue/apps.py:15-16` — global PIL side effect (dialogue before cs in INSTALLED_APPS).
  - AD: `components/utils/imageHelpers.js` (isHeic, convertHeicToJpeg), FormData RTK mutation pattern (`operatorsApi.uploadOperatorImages`).
  - FE: no client-side HEIC conversion needed — server decodes.

## Design v2 — private ImageField + presigned URLs, one endpoint, server-side upload

### ⚠️ v2 change: PRIVATE storage (audit must-fix)

v1 used public-read S3 + URLField. **Rejected by audit:** `AWS_DEFAULT_ACL='public-read'` + `AWS_QUERYSTRING_AUTH=False` (settings.py:345) = support-chat images (passports, payment screenshots) permanently public to anyone with URL. PDPA risk; URL leakage via history/referer/logs is the threat, not brute force.

**v2 storage design:**
- `Message.image = ImageField(null=True, blank=True, upload_to='', storage=ChatMediaStorage())` — NOT URLField
- New `ChatMediaStorage(S3Boto3Storage)` in `Smartenplus/media_storages.py`: `location='media/cs_chat'`, `default_acl='private'`, `querystring_auth=True` → `.url` property auto-generates presigned URL. Scoped class — existing public media untouched.
- `MessageSerializer.image_url` = SerializerMethodField → fresh presigned URL every fetch (polling path always fresh)
- Supabase `cs_messages.image_url` stores presigned URL generated at write time, **7-day TTL** (live realtime viewers always within TTL)
- **Realtime-mode history fetch switches to Django `MessageListView`** (exists already) — fresh presigned URLs for old messages. Supabase remains for live INSERT events only. Small change in `useChatRealtime.js` + `useStaffChatRealtime.js` history functions.
- ImageField bonus: future post_delete cleanup = one-liner `instance.image.delete(save=False)` (operators/signals.py:124-151 pattern) — no URL parsing.

### Endpoint flow

`POST /api/cs/messages/send-image/` (multipart: conversation, image, optional body):

auth (`_resolve_message_sender`) → **guest gate** → `process_operator_image(max_output_size=80*1024, max_dimensions=(1200, 1000, 800))` → save to `Message.image` (private S3) → reopen-check + Message.create (atomic, BEFORE broadcast) → Supabase mirror insert w/ 7-day presigned URL (service role) → backfill `supabase_id` → 201

**Guest gate (v2.1):** plain guests (email-OTP, no OTA booking) blocked. OTA convs have `Conversation.ota_booking_id` set; plain-guest convs don't:
```python
# Image send: OTA + logged-in only. Plain guests (no OTA booking) text-only.
if sender == Message.SENDER_CUSTOMER and not request.user.is_authenticated and not conv.ota_booking_id:
    return _error('IMAGE_NOT_ALLOWED', 'Image upload is available for booking-linked chats only.', status=403)
```

**Both transports converge server-side for images.** Django's Supabase insert fires postgres_changes → realtime clients receive instantly. Polling clients get fresh presigned URL via serializer. Text path byte-identical, untouched.

**Decisions (v2.1):**
- **Guests blocked from image send** — OTA + logged-in only (per spec). BE gate via `conv.ota_booking_id`; FE hides attach button (`canSendImage = Boolean(state.token) || Boolean(otaToken)` passed ChatWidget → ChatPanel). Server gate = authority; FE flag = UX only. Text chat unchanged for guests.
- 1 image/message · optional caption via body · image-only (empty body) allowed
- 20MB input max · **WebP ≤80KB output, dimension ladder (1200, 1000, 800)** — for text screenshots resolution > quality; 1200px @ lower q beats 600px @ q85 for reading booking refs
- Throttle `cs_image: 30/hour` **keyed by guest-token value or user id** (NOT IP — Thai hotels/airports share NAT IPs → false 429s). Custom `get_cache_key()` ~5 lines, IP fallback only if neither present.
- **NO S3 lifecycle rule** (v1 90-day rule dropped — DB keeps conversations forever; expiring images = broken bubbles in old disputes. 100k images ≈ 8GB ≈ $0.18/mo — cost is noise)
- **NO delete signal / no image_s3_key column yet** — verified nothing in production hard-deletes Message/Conversation (only Django-admin superuser action). Defer to when delete feature ships; ImageField makes it a one-liner then.

## Critical corrections — v1 (Plan agent) + v2 (audit)

### From v1 review (still valid)
1. **Duplicate-sync:** endpoint captures Supabase row id (`Prefer: return=representation`) → backfill `Message.supabase_id` immediately, else sync re-imports as duplicate bubble. Ms-scale race accepted.
2. **No client-side HEIC:** server decodes. FE preview = filename chip; AD may reuse `convertHeicToJpeg` for preview only.
3. **Preview fallback both sides:** `get_last_message_preview` (cs/serializers.py:44) returns '📷 Photo' for image-only (polling inbox) + AD SideList client patch (realtime).
4. **Reopen-check BEFORE Supabase insert** — broadcast is irrevocable.
5. **Spoofed image_url via RLS insert:** render-time domain allowlist in FE + AD (presigned URLs → allowlist = S3 bucket host).

### From v2 audit (new — all verified against code)
6. `MessageSerializer.fields` MUST include image field (cs/serializers.py:72-76) — else polling clients see nothing.
7. `cs/tasks.py:221-226` — `Message(...)` constructor in bulk_create MUST map image column, else race-window rows lose images permanently.
8. `MessageImageCreateView` MUST NOT copy body-required guard (MessageCreateView line 432) — image-only = empty body allowed. `body=''` valid for TextField.
9. **Guest-token cross-conversation ownership check** MUST live in shared `_resolve_message_sender()` — highest implementation risk if omitted in new view.
10. Wrap `Image.open()` in try/except `UnidentifiedImageError` → 400 not 500 (operators/utils.py:80 currently uncaught; Pillow open = de-facto content sniffing; WebP re-encode destroys polyglot payloads).
11. Register `cs_image` rate in `REST_FRAMEWORK['DEFAULT_THROTTLE_RATES']` — missing = `ImproperlyConfigured` on first request.
12. Add `pillow_heif.register_heif_opener()` to `cs/apps.py:ready()` (idempotent) — kills silent dependency on dialogue app load order.
13. AD `SideList.js:124` destructure must add image field (line 132 patch alone insufficient).
14. AD `ConversationDetail.js:80-98` — image branch BEFORE the Supabase insert at line 85.
15. Polling latency claim: ≤5s only when active; backoff extends to 30s idle (useChatPolling.js:69).
16. `insert_cs_message` refactor: `Prefer: return=representation` on new function only; 4 existing system-message call sites (cs/views.py:128, 284, 314, 353) unchanged behavior.

### Verified-fine (no action)
No circular import cs←operators · pillow_heif effectively global · FE dedup ChatWidget.js:52-58 + AD dedup ConversationDetail.js:37-42 handle own-echo (no optimistic insert → no dup) · tokens available in ChatPanel props (ChatWidget.js:337-354) · S3 host in next.config.js remotePatterns both repos (plain `<img>` anyway) · no CSP blocking · `useStaffInboxRealtime` passes raw payload (transparent, no change).

### Line-number errata

## Edge cases

| Case | Handling |
|------|----------|
| Image-only, empty body | Allowed; body=''; preview '📷 Photo' server+client |
| Caption >4000 | 400 BODY_TOO_LONG |
| Auto-reopen on image | Shared atomic helper; 429 before broadcast; orphan private S3 file on 429 accepted |
| Sender attribution | Server-derived via shared helper — spoof-proof |
| Sync back-fill | supabase_id backfilled → dedup; sync maps image_url for race window |
| Mark-read / unread | Untouched |
| Presigned URL expiry | Polling: fresh every fetch. Realtime live: 7-day TTL, always fresh. Realtime history: fetched via Django → fresh. No stale-URL path remains |
| Supabase insert fails | 201 + warn; realtime viewers see on next history load |
| HEIC iPhone | Server decodes; FE preview = filename chip |
| Corrupt/spoofed file | Pillow open fails → 400; WebP re-encode destroys polyglot payloads |
| Shared WiFi | Throttle keyed by token/user — co-located guests unaffected |
| Plain guest crafts multipart request | 403 IMAGE_NOT_ALLOWED (server gate; FE never shows button) |
| Guest logs in mid-chat | Existing login RESET flow creates new authed conv → attach button appears |

## Rollback

Each step independently deployable + backward-compatible. Rendering presence-guarded. Columns nullable additive. ChatMediaStorage scoped — existing media ACLs untouched. No data migration to unwind.

## Verification matrix

**OTA / logged-in / staff** × **realtime / polling** (+ guest negative tests):
1. Image-only send → renders both directions (realtime: instant; polling: next tick, up to 30s if idle)
0. **Guest:** attach button hidden in widget; crafted multipart POST w/ guest token (non-OTA conv) → 403 IMAGE_NOT_ALLOWED; guest text send unchanged; guest→login mid-chat → attach appears in new conv
2. Image + caption → both render
3. Text send regression — unchanged
4. Admin inbox preview '📷 Photo' + unread bump
5. Closed conv + image → reopens; 4th reopen/hr → 429 in UI
6. Reject 25MB / .gif(FE)/.exe-renamed-.png (400 not 500) / 4001-char caption; HEIC → WebP end-to-end
7. DB: single Message row, supabase_id populated, no dup after sync tick; S3 object `media/cs_chat/` ≤80KB WebP ≤1200px, **private ACL**
8. Cross-conv guest token → 403
9. **Privacy:** raw S3 URL without query params → 403 AccessDenied; presigned URL works; expired presigned → 403
10. **Throttle:** 2 devices same IP, different conversations → independent 30/hour budgets
11. Old conversation (>7d) opened in realtime mode → images load (Django history path, fresh presigned)
