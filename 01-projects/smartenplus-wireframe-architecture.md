# SmartenPlus Full Platform Wireframe Architecture
> Product positioning, DNA layers, information architecture, and emotional value extracted to [[smartenplus-product-positioning]]. This doc focuses on wireframes and page structures.

# Homepage Goal
Psychological flow: I can move around Thailand easily → Thailand feels exciting → I know where to go → I know what to do → I trust this platform → I can plan confidently

# Mobile UX Strategy
Use: horizontal snap scrolling, sticky search, large touch targets, image-first discovery, floating CTA patterns
Avoid: dense desktop grids, overloaded filters, tiny buttons

# Design System
Typography: editorial serif headlines, modern sans-serif UI, large hierarchy, minimal metadata
Color System: deep navy, tropical blue, warm ivory, muted green, soft gray

# Strategic Product Principle
SmartenPlus should NOT feel like: booking utility, tourism portal, activity marketplace
SmartenPlus should feel like: A premium Thailand travel ecosystem

The strongest moat: understanding how travelers move around Thailand

Everything should reinforce: movement, exploration, connected journeys, travel confidence, route intelligence

# Final UX Goal
Users should emotionally feel: I can confidently explore Thailand

The platform should feel: cinematic, immersive, trustworthy, premium, modern, emotionally inspiring
